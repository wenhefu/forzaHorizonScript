# 地平线6 刷分助手

用虚拟手柄在 EventLab 自定义图里自动刷技能点：自动循环「踩住油门前进 → X 重开 → A 确认 → A 开始」。
**不改任何游戏文件**，只模拟一个 Xbox 手柄。

## 一次性准备
1. 安装 **ViGEmBus 驱动**（虚拟手柄的系统驱动）。没装的话界面顶部会提示，并提供安装/修复按钮。
2. 只有想改代码或自己打包的人，才需要再装 **Python 3**。

## 怎么用（拿到 exe 的朋友）
1. 先把游戏设置为窗口模式：**设置 -> 视频 -> 亮度 -> 全屏幕：关闭**。
2. 进游戏，选好刷分用的 EventLab 图，把斯巴鲁 22B 摆在起点。
3. 打开 `Forza6Helper.exe`。
4. 默认使用 **模式一 · 刷技能点（EventLab）**。按需在界面里改这几个值：
   - **启动倒计时（秒）**：留几秒让你切回游戏
   - **刷图运行时间 / 每轮刷图时间（分钟）**：模式一里 0 = 一直刷到你点停止；模式三里它只控制每轮 EventLab 刷图时间，买车加点不计时，推荐填 `60`
   - 参数不需要保存，点 **开始** 时会读取当前输入；无效输入会自动修正并写回输入框。
5. 普通界面只显示模式一/二/三。模式四、模式五、手动按 A/B、识别一次和高级复选框都在 **显示高级/调试模式** 里。
6. 推荐让游戏保持前台，按 **Ctrl+Alt+F8** 开始/停止；也可以点 **开始**，在倒计时内 **切回地平线**。
7. 不刷了点 **停止**，或直接关窗口。

界面右侧的 **开始前准备** 会提醒三件事：
- 把加满点数的 `1998 Subaru Impreza 22B-STI` 加入“车库 -> 我的车辆”的收藏，并把当前驾驶车辆设为这台车。
- 把刷技能点 EventLab 图放进“创意中心 -> 游玩赛事 -> 我的收藏”的第一个位置。当前推荐共享代码：`890 169 683`。
- 开始前停留在暂停菜单的首个分页，也就是上方分页从“剧情 / 车辆 / 我的地平线 / 在线 / 创意中心 / 商店”开始的页面。

日志在 `logs\forza6helper.log`，界面里的 **打开日志** 可以直接查看。

## 运行模式
- **模式一 · 刷技能点（EventLab）**：不盲计时。程序在内存里截取 Forza 窗口并识别画面：开始菜单先确认光标在“开始竞赛赛事”再按 A，比赛中保持油门，结果页按 X，重开确认按 A；如果识别到暂停菜单会先按 B 返回。截图不保存到硬盘。
- **模式二 · 买车加点（先买22B）**：会打开暂停菜单，进入车辆页，进入车展，前往制造商，选择斯巴鲁 22B，走默认设计/颜色并确认购买；买完后返回车辆页，进入升级与调校里的车辆熟练度，按固定路径买到抽奖精灵，再回到购买与出售页继续循环。买车流程会用 OCR 作为二次确认来识别关键文字，例如“购买新车与二手车”“车展”“制造商”“斯巴鲁”“控制器未连接”等。
- **模式三 · 买车+刷分组合（实验）**：先跑模式二买车加点；检测到“技术点数不足/不够购买额外加成”后，自动退回自由漫游，打开暂停菜单，进入创意中心 -> EventLab -> 赛事 -> 我的收藏。随后会进入收藏赛事，选择“单人”，在“我的车辆”页打开筛选并勾选“收藏”，用 OCR+高亮确认选中 22B 后进入 EventLab 开始赛事菜单，再交给模式一刷分。界面的“每轮刷图时间”只计算 EventLab 刷图阶段，**买车加点阶段不计入时间**；推荐填 60 分钟，填 0 时使用 `config.py` 里的 `COMBO_EVENTLAB_FARM_SECONDS` 默认值。到点后不会突然停止：会等当前比赛跑完，在结算页按 A 退出，等画面回到自由漫游再按 Menu 打开暂停菜单，把控制权交回买车阶段，如此循环。
- **高级/调试里的模式四 · 前台计时（兜底）**：旧逻辑。游戏保持前台，按设定秒数跑，失焦会暂停脚本计时并尝试切回。
- **高级/调试里的模式五 · 后台尝试（实验，不保证）**：想边刷边用电脑时选它。会尝试"保持活动" + 建议你把游戏设成 **无边框窗口**。但地平线很可能失焦就暂停，**本模式不保证有效**。
- 为什么不做"强力后台"：nofocusloss 那类工具靠**注入游戏进程**，而地平线常驻在线 + 首发反作弊，**篡改进程会被永久封号**（工具作者自己也写明 multiplayer 不能用）。所以本软件坚决不注入。

## 代码结构
- `modes.py`：统一登记模式名称、默认设置、是否普通界面可见，以及对应 runner 类型。
- `settings.py`：启动时从 GUI 捕获不可变的 `RuntimeSettings`，避免隐藏控件在运行中继续改变状态。
- `app_controller.py`：集中管理虚拟手柄、runner 生命周期、开始/停止、手动按键和识别入口。
- `gui.py`：只负责窗口、输入框、按钮、日志展示和产品/调试折叠。
- `assets/prep/`：产品界面右侧“开始前准备”的三张引导截图；`build.bat` 会把 `assets` 一起打进 exe。
- `tests/test_product_shell.py`：不碰游戏、不依赖虚拟手柄的结构测试，防止产品/调试模式分层被改坏。

## 当前验证状态
- 模式一和模式二都已经经过实机流程验证，可以作为稳定基线继续开发。
- 模式二在技能点不足时会停止，不会继续误买或误加点。
- 模式三已现场验证主链路：买车加点 -> 点数不足 -> EventLab 我的收藏 -> 筛选收藏 -> 选择 22B -> 进入刷图入口。“每轮刷图时间”到点后的“等当前比赛结束 -> A 退出 -> 回暂停菜单 -> 下一轮买车”因为验证周期较长，仍建议后续用短时间配置专门验证。
- 详细 checkpoint 见 `docs\2026-05-25-stability-checkpoint.md`。

智能识别默认截图频率：
- 菜单/确认页：约 0.75 秒一次
- 刚进入比赛的前 10 秒：约 2 秒一次
- 比赛稳定行驶后：约 5 秒一次

## 自己打包成 exe（开发者）
- 在 **Windows** 上双击 `build.bat`，生成 `dist\Forza6Helper.exe` 和 `dist\README.txt`。
- 发给朋友时建议把 exe 和 README 一起打包成 zip。exe 会在启动时检查 ViGEmBus；朋友电脑仍然需要各自装一次驱动。
